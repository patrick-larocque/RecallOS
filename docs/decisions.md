# RecallOS Decisions

## Purpose
Track key product and architecture decisions over time.

---

## Decision: Local-first architecture
Status: Accepted

All core functionality must work offline after initial setup.

---

## Decision: Explicit capture only
Status: Accepted

The app will not automatically ingest all device data.

---

## Decision: Source-grounded answers
Status: Accepted

All answers must include references to source memory items.

---

## Decision: Hybrid retrieval
Status: Accepted

Combine keyword and vector search for reliability.

---

## Decision: Modular architecture from start
Status: Accepted

Use multi-module structure instead of single-module Android app.

---

## Decision: Routing layer abstraction
Status: Accepted

All local vs network decisions are centralized.
