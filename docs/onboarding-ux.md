# RecallOS Onboarding & UX

## Goal
Define the first-run flow, core interaction model, and trust surfaces for RecallOS.

## Product UX principles
- Capture-first, not chat-first
- Explicit user control
- Source-grounded answers
- Fast save, optional enrichment
- Local-first trust model

## First-run onboarding
1. Position the app as a private memory layer
2. Explain the save → understand → ask loop
3. Surface execution mode controls
4. Guide the user to first capture
5. Trigger a first successful retrieval

## Core surfaces
- Home
- Search / Ask
- Spaces
- Memory detail
- Settings
- Capture entry points

## Primary flows
### Save flow
User saves content from share sheet, note entry, import, or quick action.

### Ask flow
User asks a question over saved memory and receives an answer with source cards.

### Explore flow
User browses recent captures, spaces, and memory details.

## Trust surfaces
Every answer should show:
- execution mode
- source cards
- result type

## UX rules
- Do not force metadata at save time
- Do not hide local/network execution
- Do not center the experience on a blank chat screen
- Preserve raw content even if processing fails
