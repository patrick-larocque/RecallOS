# RecallOS Roadmap

## Purpose
Define the staged rollout from foundation to first releasable Android build.

## Phase 0 — Foundation
- repository scaffold
- modular project setup
- build logic
- version catalog
- initial docs

## Phase 1 — Core storage
- shared models
- Room schema
- file storage layer
- repository contracts

## Phase 2 — Capture
- share target
- quick add
- note entry
- image and file import
- basic voice memo

## Phase 3 — Ingestion
- OCR
- transcription
- chunking
- metadata generation
- embedding pipeline
- processing status

## Phase 4 — Retrieval and Ask
- keyword search
- vector search
- hybrid ranking
- source-grounded answers
- answer mode labeling

## Phase 5 — Spaces and Controls
- spaces
- settings
- execution modes
- privacy controls
- model/device handling

## Phase 6 — Release hardening
- testing and QA
- performance tuning
- failure handling
- release criteria

## V1 release target
A local-first Android app that can capture user-saved content, process it locally, retrieve it semantically, and answer questions with source grounding.
